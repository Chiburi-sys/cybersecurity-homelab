## Summary

A Kerberos TGS ticket request with RC4 encryption was detected for the `svc_sql` service account. The request originated from the previously compromised `j.smith` account on 192.168.50.30. This indicates an attacker is attempting to extract a crackable service account hash.

---

## Detection

- **Alert:** Kerberoasting — Suspicious TGS Request
- **Source:** Splunk (EventCode 4769)
- **Trigger:** TGS request with encryption type 0x17 (RC4) for a non-machine SPN

---

## Investigation

### Step 1: Identify the Request

```spl
index=main EventCode=4769 Ticket_Encryption_Type=0x17
  Service_Name!="krbtgt" Service_Name!="*$"
| table _time, Account_Name, Service_Name, Client_Address, Ticket_Encryption_Type
```

**Result:** `j.smith` requested TGS for `DC01/svc_sql.cybersec.local:60111` with RC4 encryption.

### Step 2: Correlate with Earlier Compromise

```spl
index=main Account_Name="j.smith" (EventCode=4624 OR EventCode=4769)
| table _time, EventCode, Source_Network_Address, Service_Name
| sort _time
```

**Result:** j.smith was compromised via password spray (SOC-001) at 10:16 AM, then used for Kerberoasting at 12:10 PM.

### Step 3: Assess the Target Account

- **svc_sql** — SQL service account with SPN registered
- Password strength: **Weak** (8 characters, dictionary word)
- Last password change: 2026-03-01
- **Risk: HIGH** — if cracked, attacker gains service account access
